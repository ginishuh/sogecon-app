# 2026-07-13 개발 로그

## 게시판 작성일·저콘텐츠 상태 마무리

- #239에서 일반 동문 게시글의 작성 시각을 `created_at`으로 저장·노출하고, 관리자 발행 글은 `published_at`을 우선하는 날짜 계약을 추가한다. (API/contract/frontend impact)
- 기존 게시글은 실제 작성 시각을 복원할 근거가 없어 null로 보존하며 Web에서는 `작성일 미확인`으로 안내한다.
- 공개 목록은 고정 글 우선 후 `published_at ?? created_at` 최신순과 ID 역순으로 안정 정렬한다.
- 게시판의 `더 불러오기`를 누르면 다음 페이지로 교체되던 동작을 누적 목록으로 바로잡고 0건·1건·다수 상태를 자동 테스트로 고정한다.

## 로그인·가입신청·첫 로그인 설정 디자인 리뉴얼

- #237에서 로그인·가입신청·비밀번호 만들기를 하나의 동문 인증 여정으로 재구성한다. (frontend impact)
- 선택 시안 2의 안내형 구조를 적용해 가입 절차를 `정보 보내기 → 사무국 확인 → 비밀번호 만들기`로 먼저 설명하고, 실제 완료 상태는 별도 화면에서 유지한다.
- 가입 신청의 장문 폼을 `동문 확인 정보`와 `연락받을 정보`로 나눠 필수·선택 항목과 다음 행동을 빠르게 찾을 수 있게 한다.
- 로그인·가입신청·활성화 화면은 공통 heading·폭·여백 문법과 Button/Input primitive를 사용하며, 모바일 메뉴와 보조 링크까지 최소 44px 조작 영역을 적용한다.
- 폼 오류는 화면 안의 연결된 live region을 기준으로 안내해 동일 오류가 inline alert와 toast에서 중복 발표되지 않게 한다.
- 기수·연락처 로컬 검증은 해당 입력으로 포커스를 이동하고 `aria-invalid`·연결된 오류 문구를 제공한다.
- Windows Chrome focused QA에서 확인한 공통 헤더 홈 링크와 푸터 지원 링크의 좁은 터치 영역을 시각 간격 변화 없이 최소 44px로 확장한다.
- PR #238 리뷰를 반영해 가입 API 오류 banner로 포커스·스크롤을 복구하고, 로컬 입력 오류에서는 이전 API 오류를 제거한다. 승인 대기 문의 링크와 로그인 보조 경로의 44px·정렬 계약, API 오류·기수·선택 메모 회귀 테스트도 함께 보강한다.
- 재리뷰 Note의 기수 입력 경계를 함께 닫아 숫자가 아닌 문자가 섞인 값을 부분 숫자로 수용하지 않게 한다.
- Ready CI에서 확인된 onboarding E2E helper를 표준 `label[for]` 연결과 접힌 선택 메모 흐름에 맞춰 갱신한다.
- 공개 라우트 30개의 고유 raw JS 청크를 합산하는 번들 게이트는 디자인 리뉴얼 이후의 정상 증가를 반영해 `1127KB`에서 `1536KB`로 조정하되 초과 시 실패하는 회귀 게이트는 유지한다. (CI impact)

## 행사 일정·상세 디자인 리뉴얼

- 공지·동문 소식 분류 버튼의 현재 선택 상태를 `aria-pressed`로 노출하고, 존재하지 않는 상세를 영문 기본 404 대신 한국어 안내와 목록 복귀 행동으로 연결한다.
- #235에서 제목 없이 한 줄로 끝나던 행사 목록을 H1·설명·예정/지난 행사 필터·가장 가까운 행사·후속 목록·다음 행동이 있는 빈 상태로 재구성한다. (frontend impact)
- Desktop은 1152px 안에서 날짜·행사 정보·정원·상태·상세 행동을 가로로 비교하고, Tablet/Mobile은 같은 정보 순서를 한 열로 reflow한다.
- 존재하지 않는 행사 상세의 404는 재시도 대기 없이 한국어 안내와 행사 목록·사무국 문의 복구 행동으로 전환한다.
- 행사 상세의 일정·장소·정원·설명·RSVP를 저널형 정보 계층과 44px 조작 계약에 맞춰 정돈한다.
- PR #236 리뷰를 반영해 공개 행사를 100건 단위로 끝까지 조회하고, 예정·진행·지난 시기 경계와 정렬을 회귀 테스트로 고정한다. 지난 행사 제목·목록 라벨, KST 연도, 잘못된 상세 ID 복구 문구도 같은 맥락으로 정렬한다.
- Windows Chrome 재검증에서 불안정했던 행사 필터의 키보드 포커스 표시는 실제 focus 상태에 브랜드 링을 적용해 선택 상태와 별도로 식별되게 한다.

## 내 정보·공개 범위 디자인 리뉴얼

- #233에서 `내 정보와 공개 범위`를 긴 관리 폼에서 프로필 확인 중심 화면으로 재구성한다. (frontend impact)
- 상단에 회원 identity와 저장 상태를 노출하고, 동문 수첩에 실제로 보일 프로필 미리보기를 제공한다.
- 공개 범위를 세 가지 설명형 radio 선택으로 바꿔 대상과 영향을 동시에 비교할 수 있게 한다.
- 연락처와 소속 정보는 기본 접힘 요약으로 두고 필요한 영역만 펼쳐 편집하며, 검증 오류가 있는 영역은 자동으로 연다.
- 이름·기수 변경 요청은 일반 프로필 저장과 분리된 모듈로 정리하고 요청 이력의 빈 상태를 명시한다.
- 모바일 44px 조작 영역, 음력 선택 터치 범위, H1→H2 heading 구조와 반응형 `min-w-0` 계약을 보강한다.
- Windows production 시각 QA에서 발견한 모바일 세로 밀도를 보정해 사진 변경을 identity의 44px 보조 행동으로 통합하고, preview·privacy·요약 행 간격을 압축한다. 최종 실측에서 저장 CTA는 `docY=793.39`, 변경 요청은 `docY=942.19`로 핵심 행동 목표를 충족한다.
- PR #234 리뷰를 반영해 file input을 반응형 숨김 부모 밖의 활성 DOM으로 이동하고 모바일 제한 안내를 복구한다. 공개 범위 카드 focus ring, 아바타 공용 URL resolver, 변경 요청 loading/empty 구분, validation 자동 펼침 및 file chooser 트리거 테스트도 함께 보강한다.
- 재리뷰의 잔여 Minor까지 반영해 변경 요청 이력 조회 실패를 빈 상태와 구분하고, 사용자 안내와 재시도 행동을 제공한다.
