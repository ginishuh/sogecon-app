# 동문 사용자 언어 원칙

일반 동문 화면의 문구는 사용자가 다음 행동을 바로 이해할 수 있는 한국어를 사용합니다.
코드에서 반복 사용하는 핵심 명칭은 `apps/web/lib/member-language.ts`와 함께 관리합니다.

## 공통 용어

| 내부·기존 표현 | 일반 동문 화면 | 사용 원칙 |
| --- | --- | --- |
| 관리자 문의 | 동문회 사무국 문의 | 문의 주체를 실제 조직명으로 안내 |
| 신규 가입신청 | 동문 가입 신청 | 대상과 행동을 함께 표시 |
| 계정 활성화 | 첫 로그인 설정 | 사용자가 수행할 행동으로 설명 |
| token, 활성화 토큰 | 계정 활성화 코드 | 입력 화면에서만 사용하고 기술 형식은 숨김 |
| VAPID, 서비스 워커, 구독 | 새 소식 알림 | 실패 시 원인 대신 복구 행동을 안내 |
| role, RBAC, `admin`, `super_admin` | 운영 권한, 담당 업무별 접근 권한 | 일반 화면에 내부 권한 식별자를 노출하지 않음 |
| RSVP | 행사 신청 | 동문이 익숙한 행동 용어 사용 |

## 오류와 상태 문구

- 일반 동문 화면의 오류는 내부 코드나 객체 문자열 대신 현재 상태와 다음 행동을 안내합니다.
- 알 수 없는 오류는 잠시 후 재시도하도록 안내하고, 계정 문제는 동문회 사무국 문의로 연결합니다.
- 일반 화면에는 VAPID, token, role, RBAC, HTTP 상태나 내부 오류 코드를 표시하지 않습니다.
- 운영자 전용 화면은 업무 수행에 필요한 상세 오류와 운영 용어를 보존할 수 있지만 API 내부 식별자를 그대로 노출하지 않습니다.
- 성공 문구는 완료된 행동을 짧고 분명하게 표현합니다.

## 화면별 기준

- 헤더·Drawer: `동문 가입 신청`, `첫 로그인 설정`, `이용 안내`
- 로그인·가입: 승인 주체는 `동문회 사무국`, 승인 후 행동은 `첫 로그인 설정`
- 문의: `동문회 사무국 문의`와 문의 가능한 범위를 함께 표시
- 알림: `새 소식 알림 켜기/끄기` 중심으로 안내하고 브라우저 내부 구현은 숨김

## 가입부터 첫 로그인까지

가입 여정은 다음 네 단계를 같은 순서와 이름으로 안내합니다.

1. 가입 정보 보내기
2. 동문회 사무국 확인
3. 비밀번호 만들기
4. 첫 로그인 완료

- 가입 신청 직후에는 아직 실행할 수 없는 로그인·비밀번호 만들기 버튼을 노출하지 않습니다.
- 승인 안내 링크로 들어오면 코드는 자동으로 적용하고 비밀번호 입력만 요청합니다.
- 코드 직접 입력은 안내 링크를 사용할 수 없을 때의 보조 경로로 제공합니다.
- 만료되거나 잘못된 링크는 사무국 문의, 이미 사용한 링크는 로그인으로 연결합니다.
- 로그인 화면은 기존 동문, 처음 방문한 동문, 승인 안내를 받은 동문의 경로를 구분합니다.
- 메뉴의 `첫 로그인 설정`은 여정 전체의 이름이고, 3단계 `비밀번호 만들기`는 해당 화면에서 수행하는 구체적인 행동입니다.
