# 2026-07-14 개발 로그

## 관리자 권한 및 삭제 확인창 E2E 후속 수정

- 문의 내역 조회를 `admin_support` 세부 권한으로 분리하고 Web 메뉴, 홈 운영 행동, 관리자 역할 UI, API 의존성, bootstrap 시드를 같은 권한 계약으로 정렬했다.
- 게시물 전용 관리자는 홈과 관리자 메뉴에서 게시물 관리만 볼 수 있고 문의 내역 API에는 접근할 수 없도록 회귀 테스트를 추가했다.
- 공통 `ConfirmDialog`에 접근 가능한 이름과 설명을 연결하고, 취소·확인 버튼의 44px 최소 높이와 닫힘 뒤 호출 요소 포커스 복귀를 적용했다.
- 삭제된 게시글 상세에 한국어 복구 화면과 게시판 복귀 행동을 추가했다.
- 관리자 게시물·문의 화면의 인증 확인 중 문구를 실제 비인증 안내와 분리했다.
- 실제 PostgreSQL에서 문의 목록이 500으로 실패하던 원인인 `support_tickets` 누락을 Alembic 마이그레이션으로 보완했다. 테스트의 `metadata.create_all()`이 가리던 스키마 드리프트를 production 계열 경로와 일치시켰다.

## 역할 및 프로필 변경 심사 접근성

- Windows visible E2E에서 확인된 관리자 역할 checkbox·저장 버튼과 프로필 변경 심사 조작의 작은 hit area 및 focus 표시 누락을 공통 44px/focus-ring 계약으로 정렬했다.
- 프로필 변경 반려 사유 textarea에 visible label을 연결하고, 상태 필터 label, 승인·반려·페이지 이동 컨트롤에도 접근 가능한 이름과 키보드 focus 표시를 적용했다.
- 관리자 프로필 변경 화면의 인증 확인 중 상태를 비인증 안내와 분리하고, 해당 컨트롤 계약을 자동화 테스트로 고정했다.
## 관리자 콘텐츠 화면의 홈 배너 권한 경계 보강

- `admin_posts` 또는 `admin_events`만 가진 제한 관리자가 목록·수정 화면에 들어갈 때
  `admin_hero` 전용 lookup을 선행 호출해 403을 반복하던 문제를 수정했다.
- 홈 배너 lookup과 토글 UI를 `admin_hero` 권한에 맞춰 조건부 활성화하고, 목록 데이터 요청도
  해당 콘텐츠 권한이 확인된 뒤 시작하도록 정리했다.
- 권한이 없을 때 lookup을 호출하지 않고, 권한이 있을 때만 호출하는 hook 회귀 테스트를 추가했다.

## 홈 배너 관리 화면의 직접 접근 권한 경계 보강

- `/admin/hero`와 `/admin/hero/[id]/edit`는 세션의 `admin_hero` 권한이 확인된 뒤에만 목록·상세 API를 조회하도록 변경했다.
- 권한 안내 화면을 보여 주면서 백그라운드에서 403 요청을 만들던 직접 접근 회귀를 자동화 테스트로 고정했다.

## 가입 신청 검증 오류 사용자 문구 보강

- 가입 신청 API의 Pydantic 422 검증 원문이 화면에 그대로 노출되지 않도록 회원 여정의 fallback 문구를 한국어 사용자 안내로 제한했다.
- 이메일·연락처 형식 검증 실패는 사용자가 확인할 입력을 바로 알 수 있는 한국어 안내로 바꾸고 회귀 테스트를 추가했다.

## 가입신청 심사 권한 경계와 조작 접근성

- `admin_roles` 권한이 없는 제한 관리자가 `/admin/members`에 직접 접근할 때 회원 목록·개수 API를 먼저 호출하지 않도록 query를 권한 확인 뒤에만 활성화했다.
- 가입신청 심사 화면의 인증 확인 중 상태를 비인증 문구와 분리하고, 신청 상태를 대기·승인·반려·활성화 완료의 한국어로 표시했다.
- 검색·상태 필터·조회·승인·재발급·반려·페이지 이동·복사 조작을 공통 44px 및 키보드 focus-ring 계약으로 정렬했다.
- 반려 사유 textarea에 visible label을 연결하고 권한 없는 선행 요청과 심사 화면 접근성 계약을 회귀 테스트로 고정했다.

## 행사 관리자 인증 상태와 조작 접근성

- 행사 목록·생성·수정 화면의 세션 확인 중 상태를 비인증 안내와 분리해 hard reload에서 잘못된 로그인 필요 문구가 순간 노출되지 않도록 했다.
- 행사 필터, 날짜 입력, 초기화, 행 수정·삭제, 페이지 이동, 생성·수정 폼과 모든 action을 공통 44px 및 키보드 focus-ring 계약으로 정렬했다.
- 모바일 수정 화면에서 action이 자연스럽게 줄바꿈되도록 하고, 필드 label·hit area·focus 계약과 인증 loading 문구를 자동화 테스트로 고정했다.

## 알림 관리자 권한 경계와 조작 접근성

- `admin_notifications` 권한을 가진 제한 관리자에게 홈 운영 업무의 알림 관리 행동을 제공해 관리자 메뉴와 진입 경로를 일치시켰다.
- 알림 관리 화면의 인증 확인 중 상태를 비인증 안내와 분리하고, 권한 확인 전에는 통계·발송 로그 API를 요청하지 않도록 query를 권한에 맞춰 지연했다.
- 발송 폼, 통계 기간, 새로고침, 로그 정리, 표시 개수 조작을 공통 44px 및 키보드 focus-ring 계약으로 정렬했다.
- 발송 로그가 0건일 때 한국어 빈 상태를 표 안에 표시하도록 보완했다.

## 홈 배너 관리자 진입 경로

- `admin_hero` 권한을 가진 제한 관리자의 홈 운영 업무에 `홈 배너 관리` 행동을 추가해 Desktop 관리자 메뉴와 진입 경로를 일치시켰다.
- 권한별 홈 행동 필터가 배너 관리 권한만 정확히 반영하고 다른 관리자 행동은 노출하지 않는 회귀 테스트를 추가했다.
- 홈 배너 목록·생성·수정 화면이 모두 같은 `admin_hero` 판정을 사용하도록 인증 상태와 권한 경계를 통일하고, 확인 중 문구와 권한 없는 API 선차단을 같은 구조로 정리했다.
- 신규·수정 공통 폼의 입력, select, 체크박스 label, 취소·저장 버튼과 breadcrumb를 44px 이상 공통 control 규칙으로 맞추고 이미지 dropzone·제거 버튼의 키보드 focus 표시도 보강했다.

## 회원 조회 관리자 진입 경로

- `admin_roles` 권한을 가진 제한 관리자의 홈 운영 업무에 `회원 관리` 행동을 추가해 Desktop 관리자 메뉴와 진입 경로를 일치시켰다.
- 권한별 홈 행동 필터가 회원 관리 권한만 정확히 반영하고 권한 밖 관리자 행동은 노출하지 않는 회귀 테스트를 추가했다.
- 회원 목록의 검색·정렬·페이지 이동·상세 진입과 상세 화면의 목록 복귀를 44px 조작 및 키보드 focus-ring 계약으로 정렬했다.
- 조회 전용 안내와 역할·회원 상태 문구를 한국어 사용자 언어로 바꾸고, 내부 권한 토큰을 화면 설명에서 제거했다.
- 회원 목록과 상세가 같은 상태 문구 변환기를 사용하도록 통일해 `active` 같은 내부 상태값이 목록에 남지 않게 했다.
- 조회 전용 상세의 정보 표도 같은 상태 formatter를 거치게 해 상단 메타와 필드 값이 모두 한국어로 일치하도록 보완했다.

## 문의 조회 관리자 진입 경로

- `admin_support` 권한을 가진 제한 관리자의 홈 운영 업무에 `문의 내역` 행동을 추가해 Desktop 관리자 메뉴와 진입 경로를 일치시켰다.
- 문의 조회 권한만 정확히 홈 행동에 반영하고 회원 관리·가입 신청 등 권한 밖 행동은 노출하지 않는 회귀 테스트를 추가했다.
- 문의 목록의 조회 개수, 검색, 새로고침 조작을 공통 44px 및 키보드 focus-ring 계약으로 정렬하고 인증·권한 확인 전 문의 API 선요청 방지를 자동화 테스트로 고정했다.

## Alembic 문의 테이블 메타데이터 정합성

- 문의 티켓 모델을 Alembic 실행 환경에서 명시적으로 등록해 `support_tickets` 테이블과 인덱스가 자동 생성 비교에서 삭제 대상으로 오인되지 않도록 했다.
- 회귀 테스트와 `alembic check`로 문의 테이블 삭제 오판이 사라진 것을 확인했다. 전체 `alembic check`에는 이번 범위 밖의 기존 인덱스 메타데이터 드리프트가 남아 있어 별도 경계로 기록했다.

## 조직도 상단 이미지 우선 로드 유지

- 공통 About 사진 컴포넌트에 이미지 우선 로드 옵션을 추가하고 조직도 첫 화면 사진에 적용해, 기존 구현의 preload 계약과 모바일 첫 렌더 LCP 특성을 유지했다.

## PR 리뷰 후속 보강

- 문의 내역 권한을 세분화하기 전 접근 가능했던 기존 `admin` 계정에 `admin_support`를 후속 Alembic revision으로 한 번 백필해, 이미 문의 테이블 revision을 적용한 환경에서도 운영 권한 공백이 없도록 했다.
- 권한 의존성과 세션 조회가 DB의 최신 역할을 쿠키 세션에 반영하도록 해 배포 중 로그인해 있던 관리자도 재로그인 없이 백필·회수된 권한을 사용하도록 했다.
- 조직도와 같은 첫 화면 사진을 쓰는 역대 회장단·역대 원우회장에도 이미지 우선 로드를 적용하고 페이지 단위 회귀 테스트로 고정했다.
- Alembic은 모든 모델이 등록된 공통 `Base.metadata`를 명시적으로 사용하고 문의 테이블과 기존 핵심 테이블 등록을 함께 검사하도록 테스트를 강화했다.
- 관리자 화면의 loading·session error·unauthorized 문구를 공통 `AdminAuthState`로 통일해 로그인 필요 상태와 세션 조회 실패를 구분했다.
- 로컬 E2E 캡처·PID가 저장되는 `.artifacts/`를 ignore해 대용량·머신 종속 증거 파일이 저장소에 섞이지 않도록 했다.
